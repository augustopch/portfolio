Real-time graphics

Music Visualizer

Kaleidoscope — a local, capture-based WebGL visualizer with six engines that folds live audio into projected geometry, beat-locked and driven by a Wii remote.

Role  Design & build· Stack  WebGL, Web Audio, Gamepad API· Code ↗· Live ↗

Why it exists

The first version pulled track data from a streaming API. That path closed when the provider tightened its terms — a clean reminder that anything built on someone else’s API is built on rented land. So I rebuilt it the way it should have been: no API, no account, no dependency. It captures whatever audio is already playing on the machine and turns it into visuals locally.

Approach

The browser captures system or tab audio, Web Audio analyses it in real time, and WebGL renders it across six switchable engines — kaleidoscope, tunnel, plasma, spectrum, starfield, fractal. It’s tuned for techno and psytrance on a projector: motion is beat-locked rather than strobing, so it moves with the music instead of flickering against it.

  • Six real-time engines, hot-switchable, with auto-cycle and auto-evolve.
  • Wii remote control over Bluetooth — wave for energy, tilt to spin, buttons for engine and intensity.
  • Runs fully local: nothing recorded, nothing uploaded.

Designed for mixed neurotypes

A room full of people doesn’t share one tolerance for visual intensity. Flash guard is on by default, an Intensity slider and a one-key “calm it down” let anyone dial the whole thing back instantly, and it avoids strobing by design. It’s explicit that this isn’t a photosensitive-epilepsy guarantee — the honest framing matters more than a false claim of safety.

What it demonstrates

Real-time graphics and audio DSP, live hardware input over Bluetooth, and a bias toward local-first tools that don’t depend on a vendor’s goodwill. It’s the side of my work that builds for the room and the craft — and it pairs with the Smart Band Controller as the physical, gesture-driven end of what I make.
