Molecular

An interactive series that teaches governed AI orchestration by turning each topic into a molecule — atoms are concepts, bonds are dependency and control relationships.

Role  Design & build· Stack  HTML, JS, generator architecture· Code ↗· Live ↗

The problem

Governed AI orchestration is a web of dependency and control relationships — the model interprets, the system validates, deterministic tools execute, logs prove. Most teaching material flattens that into prose, which loses exactly the structure that matters. I wanted a way to see how the pieces constrain each other.

Approach

Each of the twenty topics is a 3D molecule. Atoms are concepts; bonds encode whether one concept depends on, or governs, another. You select an atom, read it, answer a recall check, and mark it learned — a focused way to study system design rather than skim it.

  • Concepts as atoms, control and dependency relationships as bonds.
  • Per-atom recall checks with local progress tracking.
  • Self-contained pages — no external dependencies, works offline.

Architecture

The twenty pages aren’t hand-authored. They’re produced by a generator: a shared engine template, one typed data module per topic, and a build script that assembles them. Adding or correcting a topic means editing structured data, not copy-pasting HTML — which is the only way twenty consistent, maintainable pages stay consistent and maintainable.

engine template · typed per-topic data modules · build script

What it demonstrates

Two things at once. Command of the subject — teaching a system is the real test of understanding it. And engineering discipline — a data-driven generator rather than twenty duplicated files. It’s also where I practise translating dense technical material into something a non-technical reader can follow.
